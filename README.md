# Sistemas Distribuidos

Proyectos desarrollados para la materia de Sistemas Distribuidos, implementando arquitecturas cliente-servidor y comunicaciГіn mediante sockets.

## рҹ“Ғ Proyectos

### 1. Multihilo
Juego en consola Java con sistema multihilo para gestiГіn concurrente.

**CaracterГӯsticas:**
- InputThread: Entrada de usuario
- GameLoop: LГіgica principal
- RenderThread: Renderizado en consola
- WorldClock: Control del tiempo del mundo

### 2. Cliente-Servidor
Juego en consola Java con arquitectura cliente-servidor utilizando **sockets TCP**.

**CaracterГӯsticas:**
- **Servidor**: Gestiona el estado del juego, acepta mГәltiples clientes
- **Cliente**: Conecta al servidor, envГӯa comandos, recibe actualizaciones
- **ComunicaciГіn**: Protocolo TCP/IP
- **Puerto**: 12345

**TecnologГӯas:**
- Java 17
- Maven
- Sockets TCP (java.net)
- JSON (org.json.simple)
- Multihilo

### 4. MultiplesClientes-MultiplesServidores
Sistema avanzado con **servidor de matchmaking** y **mГәltiples servidores de juego** utilizando sockets TCP.

**CaracterГӯsticas:**
- **MatchmakingServer**: Servidor central (puerto 12345) para crear, listar y unirse a partidas
- **Servidores de Juego**: Cada partida en servidor independiente (puertos 50001-50100)
- **Puerto DinГЎmico**: Los servidores de juego se crean automГЎticamente
- **Sistema de Combate**: Combate individual por jugador
- **SincronizaciГіn**: ConcurrentHashMap, AtomicInteger, volatile, synchronized

**Protocolo de Matchmaking:**
- `CREATE:{username}:{gameName}:{maxPlayers}` - Crear partida
- `LIST` - Listar partidas
- `JOIN:{port}` - Unirse a partida

**TecnologГӯas:**
- Java 17
- Maven
- Sockets TCP (java.net)
- JSON (org.json.simple)
- ScheduledExecutorService

### 5. ObjetosDistribuidos
Juego en consola Java con **Objetos Distribuidos** utilizando **RMI (Remote Method Invocation)** y sockets TCP.

**CaracterГӯsticas:**
- **RMI Broker**: Puerto 1099 para invocaciГіn de mГ©todos remotos
- **RemoteMatchmaking**: Objeto remoto para gestiГіn de partidas
- **Sistema de Matchmaking**: Servidor central (puerto 12345)
- **MГәltiples Servidores de Juego**: Puertos dinГЎmicos (50001-50100)
- **Sistema de Combate**: Combate individual por jugador
- **SincronizaciГіn**: ConcurrentHashMap, AtomicInteger, volatile, synchronized

**Dos Modelos de ComunicaciГіn:**
| Modelo | DescripciГіn | Puerto |
|--------|-------------|--------|
| **Sockets TCP** | ComunicaciГіn texto/JSON | 12345 (matchmaking), 50001-50100 (juego) |
| **RMI** | Objetos distribuidos con invocaciГіn de mГ©todos remotos | 1099 (broker) |

**TecnologГӯas:**
- Java 17
- Maven
- Java RMI (java.rmi)
- Sockets TCP (java.net)
- JSON (org.json.simple)
- Lanterna (interfaz grГЎfica)
